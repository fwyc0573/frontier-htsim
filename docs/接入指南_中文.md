# collective-sim 中文接入指南（简版）


## 1) 输入和输出

输入最少需要：
- `cluster`: `servers`, `gpus_per_server`
- `parallelism`: `tp/cp/dp/ep`
- `topology`: `kind`（支持 `rail/rail3/fattree/fattree3`）及参数
- `collective`: `kind`, `tensor_bytes`, `domain_dims`, `placement_order`

输出核心字段：
- `predicted_time_ms`
- `breakdown.network_ms`
- `breakdown.intra_server_ms`
- `breakdown.combined_ms`

## 2) 先跑通

```bash
python examples/integration/smoke_regression.py
```

若结果里所有 case 的 `ok=true`，说明环境可用。

## 3) 最小调用示例

```python
from collective_sim_core import predict_collective_time

scenario = {
    "cluster": {"servers": 8, "gpus_per_server": 8},
    "parallelism": {"tp": 8, "cp": 1, "dp": 8, "ep": 1},
    "topology": {"kind": "rail", "spines": 8, "servers_per_tor": 8, "paths": 8},
    "collective": {
        "kind": "allreduce",
        "tensor_bytes": 268435456,
        "domain_dims": ["DP"],                      # 必填
        "placement_order": ["TP", "CP", "DP", "EP"]  # 必填
    },
    "intra_server": {"model": "legacy_fabric"}
}

res = predict_collective_time(scenario, repo_root="/data/collective-sim")
print(res["predicted_time_ms"])
```

## 4) 固定场景 + 动态 collective（推荐）

对方通常不知道“全部 collective 列表”，所以推荐：
- 固定 `cluster/topology/parallelism/intra_server`
- 每次按事件动态填 `collective`

```python
from copy import deepcopy
from collective_sim_core import predict_collective_time

BASE = {
    "cluster": {"servers": 16, "gpus_per_server": 8},
    "parallelism": {"tp": 8, "cp": 1, "dp": 16, "ep": 1},
    "topology": {"kind": "rail", "spines": 8, "servers_per_tor": 8, "paths": 8},
    "intra_server": {"model": "legacy_fabric"},
}

def predict_for_event(event):
    scenario = deepcopy(BASE)
    scenario["collective"] = {
        "kind": event["kind"],
        "tensor_bytes": int(event["tensor_bytes"]),
        "domain_dims": list(event["domain_dims"]),
        "placement_order": list(event["placement_order"]),
    }
    return predict_collective_time(scenario, repo_root="/data/collective-sim")
```

也可以直接从内置 `scenario_profile` 加载固定场景（示例：`h800_fattree`）。
当前内置 profile 覆盖常见 8-GPU SXM 服务器：`h100_*`、`h800_*`、`a100_*`、`a800_*`：

```python
from dataclasses import asdict
from copy import deepcopy
from collective_sim_core import get_scenario_profile_builder, predict_collective_time
from collective_sim_core.schema import CollectiveConfig, ParallelismConfig

builder = get_scenario_profile_builder("h800_fattree")
base_scenario = builder(
    servers=16,
    parallelism=ParallelismConfig(tp=8, cp=1, dp=16, ep=1),
    # 先放一个占位 collective，后面每次事件再覆盖
    collective=CollectiveConfig(
        kind="allreduce",
        tensor_bytes=1,
        domain_dims=("DP",),
        placement_order=("TP", "CP", "DP", "EP"),
    ),
)

BASE = asdict(base_scenario)

def predict_for_event(event):
    scenario = deepcopy(BASE)
    scenario["collective"] = {
        "kind": event["kind"],
        "tensor_bytes": int(event["tensor_bytes"]),
        "domain_dims": list(event["domain_dims"]),
        "placement_order": list(event["placement_order"]),
        # 可选:
        # "allgather_model": "hierarchical",
        # "reducescatter_model": "hierarchical",
    }
    return predict_collective_time(scenario, repo_root="/data/collective-sim")
```

## 5) NVLink 模型使用规则

启用机内模型：

```python
scenario["intra_server"] = {
    "model": "nvlink_analytic",
    "nvlink_one_way_bw_GBps": 450.0,  # 单向带宽（GB/s）
    "nvlink_latency_us": 0.5,
    "nvlink_efficiency": 0.8,
}
scenario["collective"]["exclude_intra_server"] = True  # 必须
```

说明：
- `nvlink_one_way_bw_GBps` 是单向带宽（GB/s）
- 当 `model=nvlink_analytic` 时，必须 `exclude_intra_server=True`

## 6) tensor_bytes 的含义

`tensor_bytes` 对不同 collective 的语义**不同**，与 `htsim_runner.py` 保持一致：

| `kind` | `tensor_bytes` 含义 | 每 rank 通信量 |
|---|---|---|
| `allreduce` | 总张量大小 `T` | `2*(n-1)/n * T` |
| `allgather` | **每 rank 的输入大小** `x`（输出为 `x*n`） | `(n-1)*x` |
| `reducescatter` | **每 rank 的输出大小** `x`（输入为 `x*n`） | `(n-1)*x` |
| `alltoall` | 每 rank 的总发送量 `T`（每 peer = `T/n`） | `(n-1)/n * T` |
| `p2p` / `t2g` | 直接传输大小 | — |

> **注意：** `allgather` 和 `reducescatter` 填的是 per-rank（非总量）大小。
> 例如 TP=8 的 allgather，激活总大小为 1 GiB，应填 `tensor_bytes = 128 MiB`（1 GiB / 8）。

## 7) 网络参数配置

`network` 字段可选，默认值适配 400G 集群：

```python
scenario["network"] = {
    "linkspeed_mbps": 400000,  # 单链路带宽（Mbps），默认 400 Gbps
    "mtu": 9216,               # MTU（bytes）
    "q": 256,                  # 队列深度（packets）
    "cwnd": 64,                # 拥塞窗口（packets）
}
```

如需模拟 800G NIC 集群，将 `linkspeed_mbps` 改为 `800000`。

## 8) 常见报错

- `collective.domain_dims is required`：没传 `domain_dims`
- `collective.placement_order is required`：没传 `placement_order`
- `nodes mismatch`：`tp*cp*dp*ep != servers*gpus_per_server`
- `collective.p2p_direction must be one of ...`：`kind=p2p` 时方向配置非法（应为 `0->1` / `1->0` / `bidir`）
